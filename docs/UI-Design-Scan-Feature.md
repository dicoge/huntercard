# 相機掃描功能 - UI/UX 設計規範
# Camera Scan Feature - UI/UX Design Specification

## 1. 設計總覽

### 1.1 產品願景
讓用戶能夠透過相機掃描 hololive 卡牌，快速辨識並查看價格資訊，打造流暢的查價體驗。

### 1.2 設計目標
- **直覺性**：用戶一眼就知道如何操作
- **速度**：3 秒內完成掃描並顯示結果
- **可靠性**：在各種光線環境下都能正確識別

## 4. 交互設計

### 4.1 掃描流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  打開    │ ──→ │  相機    │ ──→ │  掃描    │ ──→ │  顯示    │
│  掃描頁  │     │  載入    │     │  處理中  │     │  結果    │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
    0ms          500ms           1000ms           2000ms
```

### 4.2 狀態指示

| 狀態 | 視覺表現 | 說明 |
|------|----------|------|
| 準備中 | 灰色掃描框 | 等待用戶操作 |
| 掃描中 | 動畫掃描線 | 正在處理圖像 |
| 識別中 | 載入動畫 | 正在匹配數據 |
| 成功 | 綠色邊框 + 價格顯示 | 找到匹配 |
| 無匹配 | 橙色提示 + 建議列表 | 顯示相似卡片 |
| 失敗 | 紅色提示 | 錯誤訊息 |

### 4.3 手勢交互

| 動作 | 效果 |
|------|------|
| 點擊掃描按鈕 | 觸發掃描 |
| 點擊閃光燈圖標 | 切換閃光燈 |
| 點擊鏡頭圖標 | 切換前後鏡頭 |
| 滑動結果卡片 | 切換相似卡片 |
| 點擊結果卡片 | 查看詳情 |
